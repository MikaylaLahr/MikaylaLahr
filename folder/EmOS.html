Final Project: Weather Map

The objective of this project is to design and test a real time weather map using a Raspberry Pi, piTFT screen, and a 32x32 LED Matrix. An API from Open Weather Map is used to collect temperature and humidity weather data for 17 locations in New York State. The collected weather data is then displayed on the piTFT screen and the LEDs corresponding to each desired location are assigned a color based on the collected data.

The components of this project include a Raspberry Pi, piTFT screen, and a 32x32 LED Matrix. An interactive weather map will be created to display real time weather data. The numerical data for temperature and humidity, as well as the local weather data for temperature and precipitation will be displayed on the piTFT. In addition, specific LEDs on a 32x32 LED Matrix will be assigned to 17 different locations in New York State. Each LED will be assigned a color that will correspond with the temperature and humidity. When the system is powered on there is a start screen followed by four different user options. These options include:

  • Relative Temperature: Displays temperature data for all of the selected locations on the piTFT along with a scale legend. Assigns LED colors based on the highest and lowest temperature values in New York State.
  • Relative Humidity: Displays humidity data for all of the selected locations on the piTFT along with a scale legend. Assigns LED colors based on the highest and lowest humidity values in New York State.
  • Cities and Towns: Displays list of all selected cities and towns. When a location is chosen, the LED corresponding to the location turns on and the temperature and precipitation data for the selected location is displayed on the piTFT along with a weather icon.
  • Temperature: Displays temperature data for all of the selected locations on the piTFT along with a predefined scale legend. Assigns LED colors based on the set scale for temperature values (0 degrees F to 100 degrees F).

Lab 4: PreemptRT Kernel Patch

The objective of Lab 4, is to explore the PreemptRT kernel patch to investigate performance improvements. The lab begins by using simple C-based signal generation techniques on the current Raspbian system. We will then build a new kernel using Raspbian 6.1.93 source and the PreemptRT patch version 6.1.92. Once the patched kernel is installed, we will use Real-Time techniques provided by the patch for signal generation. The process includes installing utilities for managing GPIO in C, implementing and measuring a square wave signal in C, then installing the new kernel by downloading the Linux source, applying the PreemptRT patch, and compiling the kernel. After booting the new kernel, we will convert the system to real-time code and test its performance. Throughout the lab, various packages like Cyclictest, Perf, pigpio, and the PreemptRT patch will be installed, and system performance will be verified using tools like Perf, Cyclictest, and an oscilloscope or PiScope. The lab aims to provide hands-on experience in altering a Linux kernel, which is valuable knowledge for embedded systems.

Lab 3: Robot Platform for R-Pi

The objective of Lab 3, is to focus on adding output capabilities to the Raspberry Pi by exploring various attached components. This includes building a robot platform for the Pi and investigating robot applications. We will integrate these applications with piTFT feedback and evaluate their performance. A key goal of the lab is to develop reusable and modular code that can be applied in future steps. Before starting the coding process, careful software design will be essential to map operations into Python functions that can be reused later. The lab report will include the design of these software components to demonstrate code reusability. Ultimately, the goal is to create an autonomous robot controlled by an application, allowing it to navigate within its environment while displaying status and control information.

Lab 2: Evaluate the Performance of the Video Player

The objective of Lab 2, is to enhance the Raspberry Pi setup by adding external buttons to expand the control functions of the video_control.py from Lab 1. We will also compare the performance of the video_test.py program from Lab 1 with its modified versions. Additionally, we will develop methods for evaluating the performance of applications on the Raspberry Pi, focusing on performance tuning. Finally, we will create test programs using the PyGame library and develop them into control panels for animated videos.

Lab 1: Control Video Playback

The main objective of this lab is to assemble the ECE5725 lab kit, which included components such as the Raspberry Pi 4 Model B, piTFT display, power adapter, breakout cables, and other essential peripherals. The first step involved installing the Raspbian Linux kernel and configuring the Raspberry Pi for optimal performance, including setting up drivers for the piTFT touchscreen display. Next, software was installed to enable video and audio playback on the piTFT. After setting up the hardware, Python scripts were written to control video playback on the piTFT screen using physical buttons. This required learning how to use the GPIO pins on the Raspberry Pi to detect button presses and map them to playback controls like pause, rewind, and fast-forward. The lab also involved developing a FIFO to send commands to the running media player instance. The final task was to integrate all the scripts into a bash script, automating the launch of the video player and control functions for a seamless hands-on experience with both hardware and software integration.
